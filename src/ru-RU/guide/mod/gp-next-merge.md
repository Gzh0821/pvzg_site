# Правила слияния

Одна из самых частых ошибок новичков - скопировать весь исходный JSON и изменить в нем только несколько строк.

В GP-Next это обычно не лучший подход, потому что после обновления игры ваша старая копия может перезаписать новые поля, добавленные игрой.

## Главное правило

**Пишите только те поля, которые действительно хотите изменить.**

GP-Next выполняет глубокое слияние вашего JSON с исходными данными игры.
Поля, которые вы не указываете, по возможности сохраняются без изменений.

## Типы

## Features

Например:

- `PlantFeatures`
- `ZombieFeatures`
- `StoreCommodityFeatures`
- `WorldmapFeatures`
- `MintObtainRoute`

Эти файлы не сливаются по индексам массива. GP-Next находит записи по идентифицирующим полям.

### Распространенные идентифицирующие поля

- большинство файлов Features: `CODENAME`
- `MintObtainRoute`: `Family`
- `StoreCommodityFeatures.Plants / Upgrade`: `CommodityName`

### Особенность `StoreCommodityFeatures`

Это не один плоский массив. Внутри несколько параллельных секций:

- `Plants`
- `Upgrade`
- `Gem`
- `Coin`
- `Zen`

При этом:

- `Plants` / `Upgrade` сливаются по отдельным записям
- `Gem` / `Coin` / `Zen` обычно заменяются как целые массивы

## Objects

Например:

- `PlantProps`
- `ZombieProps`
- `PlantAlmanac`
- `ZombieAlmanac`

Эти файлы обычно сопоставляются так:

- открывается массив `objects`
- ищется запись, у которой `aliases[0]` совпадает с указанным вами alias

То есть для редактирования растения обычно используется его первый alias.

## Levels

Файлы уровней - исключение.

`levels/*.json` обычно обрабатываются как **полная замена файла**, а не как обычное глубокое слияние отдельных полей.

## Массивы

Здесь ошибиться проще всего:

**В GP-Next массивы по умолчанию заменяются целиком. Они не сливаются поэлементно по индексам.**

Например, если вы хотите изменить:

- набор зомби вроде `Basic_Zombie`
- список разблокированных растений `PLANTS`
- `SEEDCHOOSERDEFAULTORDER`

то нужно записывать **полный новый массив**, а не только тот элемент, который вы хотите добавить.

## Пример

Изменение только стоимости солнца и перезарядки у Peashooter:

```json
{
  "objects": [
    {
      "aliases": ["peashooter"],
      "objclass": "PlantProperties",
      "objdata": {
        "SunCost": 50,
        "Cooldown": 2
      }
    }
  ]
}
```

Преимущество такого формата:

- он не перезаписывает весь `PlantProps`
- затрагивает только `peashooter`
- меняет только `SunCost` и `Cooldown`

## Когда подходит полная замена

Полная замена обычно подходит, если:

- вы редактируете массив
- вы заменяете уровень целиком
- вы меняете некоторые полные списки магазина

## Когда лучше частичное изменение

Лучше держать патч как можно меньше, если вы меняете:

- характеристики одного растения
- характеристики одного зомби
- одну запись текста в альманахе
- цену одного предмета в магазине

## Практичный рабочий процесс

1. Проверьте реальную структуру нужной записи на странице **Data**
2. Извлеките только минимальный набор нужных полей
3. Оформите это как отдельный патч
4. Перезагрузите и снова проверьте результат на странице Data
